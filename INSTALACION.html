1

Requisitos del servidor

  • PHP 8.0 o superior
  • MariaDB 10.5+ (o MySQL 8+)
  • Apache con mod_rewrite habilitado
  • Extensiones PHP: pdo_mysql, mbstring, fileinfo
ℹ️ En hosting compartido (cPanel) todas estas extensiones suelen estar habilitadas por defecto.
2

Subir archivos al hosting

Sube la carpeta mi-boutique/ completa al directorio raíz de tu hosting:

  • Conéctate vía FTP (FileZilla, Cyberduck) o usa el Administrador de archivos de cPanel.
  • Coloca la carpeta dentro de public_html/, quedando así: public_html/mi-boutique/
  • Verifica que index.php esté en la raíz del proyecto.
3

Crear la base de datos en cPanel

  1. Entra a cPanel → MySQL Databases.
  2. En "Create New Database" escribe un nombre y haz clic en Create Database.
  3. En "MySQL Users" crea un usuario nuevo con contraseña segura.
  4. En "Add User To Database" selecciona usuario y BD → ALL PRIVILEGES → Make Changes.
⚠️ Importante: Los nombres reales tendrán el prefijo de tu cuenta cPanel. Anótalos bien y actualiza config/db.php.
4

Importar el schema SQL

⚠️ Usa el archivo database/schema_hosting.sql, NO el schema.sql. El de hosting NO incluye CREATE DATABASE (prohibido en hosting compartido).
  1. Abre phpMyAdmin desde cPanel.
  2. Selecciona tu base de datos en el panel izquierdo.
  3. Haz clic en la pestaña Importar.
  4. Selecciona database/schema_hosting.sql y haz clic en Continuar.
✓ Si el import fue exitoso verás: configuracion, usuarios, productos, ventas, detalle_ventas, promociones, clientes, movimientos_inventario.

Generar contraseñas iniciales (seed)

El SQL inserta un placeholder en password_hash. Debes ejecutar el seeder para generar los hashes bcrypt reales:

Opción A — Desde terminal / SSH

php database/seed_passwords.php

Opción B — Desde el navegador (sin SSH)

Visita temporalmente:

https://tu-dominio.com/mi-boutique/database/seed_passwords.php
⚠️ Elimina seed_passwords.php inmediatamente después de ejecutarlo.

Módulo Android / App híbrida

Todos los endpoints en /api/ devuelven JSON puro. La app móvil debe:

POST
/api/auth.php?action=login — Autenticación. Body: {"email":"…","password":"…"}. Devuelve cookie de sesión.
GET
/api/productos.php?action=by_sku&sku=XXX — Buscar producto por SKU (escaneo QR).
GET
/api/productos.php?q=texto — Búsqueda de productos por texto.
POST
/api/ventas.php?action=confirmar — Confirmar venta. Transacción ACID completa.
GET
/api/reportes.php?action=resumen&periodo=mensual — Dashboard financiero.
📷 Escaneo QR: Usar BarcodeDetector API en WebView (Chrome 83+) o librería nativa.

🍪 Sesión: Enviar credentials: 'include' en cada petición Fetch.

Notas técnicas

SKU automáticoGenerado de Marca+Modelo+Color+Talla. Misma combinación = mismo SKU. El stock se acumula automáticamente.
Comisión Clip3.6% × (1 + 16% IVA) = 4.176% del total. Desglosado en reportes y tickets.
Contraseñasbcrypt cost 12. Nunca en texto plano. Recuperación por token 64 chars con TTL de 1 hora.
Sesión seguraHttpOnly · SameSite=Strict · Secure auto en HTTPS · TTL 24 horas.
Transacciones ACIDToda venta usa beginTransaction / commit / rollBack. Si falla cualquier paso, todo se revierte.
Historial inmutabledetalle_ventas guarda costo y precio históricos. Cambios futuros al catálogo no afectan reportes pasados.
Cero dependenciasPHP Vanilla 8+. Sin Composer, sin frameworks, sin librerías externas.
PortabilidadURL auto-detectada del servidor. Solo cambiar config/db.php al migrar a otro servidor.
