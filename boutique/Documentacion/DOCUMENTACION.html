Descripción general

Panel de administración web — pos.php y vistas relacionadas

El sistema completo es un panel web accesible desde cualquier navegador de escritorio o móvil. Está diseñado para el personal con acceso a computadora y para los dueños que necesitan gestionar todo el negocio.

🌐 URL: https://tu-dominio.com/mi-boutique/
El archivo principal del POS de escritorio es views/pos.php. La interfaz usa un layout de sidebar + contenido principal.

Cualquier cambio en los archivos del servidor (vistas, API, lógica) se refleja de inmediato en todos los usuarios conectados, tanto en el sistema web como en la app.

Módulos del sistema

Resumen del día y del mes: ventas por método de pago, productos con stock bajo y últimas transacciones.

Punto de Venta — Flujo completo

Layout de dos columnas: búsqueda a la izquierda, carrito a la derecha

1. Buscar producto

  • Campo de texto: SKU exacto o cualquier parte del nombre, marca, color o talla.
  • Si el SKU coincide exactamente → muestra la tarjeta del producto directo.
  • Si no → lista de resultados; clic en uno lo selecciona.
  • Botón "Cámara QR" abre la cámara web para escanear el código del producto.
  • Selector de promoción activa si hay alguna configurada y vigente.

2. Carrito

  • Botones + / − para ajustar cantidades (respeta el stock).
  • Botón ✕ para quitar un artículo.
  • Resumen automático: Subtotal / Descuento / TOTAL.

3. Opciones de cobro

MétodoDatos que pideResultado
💵 Efectivo Correo del cliente (opcional para ticket) Registra venta, descuenta stock, genera folio
💳 Clip Folio del voucher de la terminal (obligatorio) + correo (opcional) Igual que efectivo + guarda folio Clip y comisión 3.6%+IVA
📦 Apartar Nombre, anticipo, vigencia (ver sección Apartados) Guarda apartado, limpia carrito, no descuenta stock

Apartados

Reserva de prendas con anticipo — compartido entre sistema web y app

⚠️ Los apartados no descuentan stock automáticamente. La prenda debe separarse físicamente en tienda. Cuando la clienta regresa y liquida, la venta final sí descuenta el stock al procesarla por el POS.

Flujo para crear un apartado

1
Agregar prendas al carrito

Igual que una venta normal. Se pueden apartar una o varias prendas al mismo tiempo.

2
Tocar "📦 Apartar prendas"

Botón amarillo en el carrito, debajo de "Cobrar con Clip". Abre el modal de apartado.

3
Llenar los datos

Nombre de la clienta (obligatorio) · Teléfono (opcional) · Monto del anticipo que paga hoy · Vigencia en días (default 30) · Notas. El modal calcula en tiempo real: Total / Anticipo / Resta por pagar.

4
Confirmar y separar la prenda

Se genera un folio APT-XXXXXX, el carrito se limpia y la prenda se separa físicamente en tienda.

5
Cuando la clienta regresa

Procesar el pago del saldo como venta normal (efectivo o Clip). Luego abrir el apartado desde la lista y tocar "✓ Marcar completado".

Estados de un apartado

ActivoVigente, esperando que la clienta regrese
VencidoPasó la fecha límite sin acción
CompletadoLiquidado y entregado a la clienta
CanceladoCancelado manualmente

Ver y gestionar apartados (sistema web)

El botón "📦 Apartados" en la barra superior del POS abre un modal con la lista de todos los apartados activos. Cada tarjeta muestra: nombre, folio, SKUs de las prendas, total, anticipo, resta y fecha de vencimiento. Al hacer clic en una tarjeta se ve el detalle completo con opciones para completar o cancelar.

¿Qué es la aplicación?

App híbrida: WebView nativa que carga views/pose.php

La app de Android no es una app nativa pura ni una PWA independiente. Es una WebView (un navegador embebido dentro de la app) que carga la página pose.php del servidor. El contenido y la lógica viven en el servidor; la app solo es el contenedor que lo muestra en pantalla completa y aporta acceso a la cámara del teléfono.

✅ Ventaja clave: cualquier cambio en pose.php se refleja automáticamente en la app sin necesidad de actualizar el APK ni publicar una nueva versión en la tienda.

pose.php es un archivo autónomo: contiene todo el HTML, CSS y JavaScript en un solo archivo, sin depender de assets/js/app.js ni de otros recursos externos, lo que garantiza que la WebView lo cargue de forma confiable.

🏗️

Arquitectura

Cómo se comunican la app, el servidor y la base de datos

┌─────────────────────────────────────┐ │ App Android (APK) │ │ WebView → pose.php │ │ Bridge: AndroidApp.abrirCamaraNativa() │ └──────────────┬──────────────────────┘ │ HTTPS ┌──────────────▼──────────────────────┐ │ Servidor (tu-dominio.com) │ │ views/pose.php │ │ api/*.php │ └──────────────┬──────────────────────┘ │ PDO ┌──────────────▼──────────────────────┐ │ Base de datos MariaDB │ └─────────────────────────────────────┘

Bridge nativo — AndroidApp

El objeto AndroidApp es expuesto por la WebView de Android al JavaScript de la página. Actualmente expone:

if (typeof AndroidApp !== "undefined") { AndroidApp.abrirCamaraNativa(); // Abre el escáner del teléfono. // Cuando lee un código, devuelve el SKU // al campo de búsqueda automáticamente. }

Si AndroidApp no existe (p. ej. al abrir pose.php en un navegador normal), la app cae automáticamente al escáner web con la API BarcodeDetector del navegador.

⚠️ La app requiere conexión a internet en todo momento. No hay modo offline ni caché local de datos. Sin conexión, las búsquedas y los cobros no funcionan.

Pantalla de Login

Primera pantalla si no hay sesión activa

  • Campo de correo electrónico y contraseña (con botón ojo para mostrar/ocultar).
  • Botón "Iniciar sesión" — llama al API y arranca la sesión.
  • Enlace "¿Olvidaste tu contraseña?" — muestra el formulario de recuperación sin salir de la app; el servidor envía el enlace de restablecimiento al correo.
  • La sesión persiste mientras la app esté abierta (cookie de sesión del servidor, 24 horas). Al expirar, regresa automáticamente al login.

Pestaña Buscar

Búsqueda de productos por SKU, nombre o cámara

🔍BUSCAR
🛒CARRITO
📦APARTADOS

Barra de búsqueda

  • Campo de texto con teclado optimizado para código de barras.
  • Búsqueda en tiempo real después de 350 ms de pausa al escribir.
  • Busca por SKU exacto, o en nombre, marca, color y talla simultáneamente.

Botón 📹 Cámara

  • En la app Android: activa el escáner nativo del teléfono (más rápido y confiable). Lee QR y códigos de barras Code 39.
  • En navegador web: abre la cámara trasera del dispositivo y detecta el código con BarcodeDetector.
  • Al leer un código, el SKU se pone en el campo y lanza la búsqueda automáticamente.

Resultados

  • SKU exacto → muestra la tarjeta del producto directo.
  • Coincidencias parciales → lista de resultados; toca uno para seleccionarlo.
  • La tarjeta del producto muestra: nombre, SKU, stock disponible y precio. Botón "+ Agregar" lo añade al carrito.

Selector de promoción

Si hay promociones activas y vigentes, aparece un selector. Al elegir una, el descuento se aplica automáticamente al total del carrito.

Artículos en el carrito

Pestaña Apartados

Crear y gestionar reservas de prendas con anticipo

🔍BUSCAR
🛒CARRITO
📦APARTADOS

Crear un apartado

El apartado no tiene buscador propio. Las prendas se seleccionan desde la pestaña Buscar y se agregan al carrito como en una venta normal. Luego en el carrito se toca "📦 Apartar prendas".

1
Buscar → Carrito

Agrega las prendas a apartar igual que si fuera una venta normal.

2
Tocar "📦 Apartar prendas"

Sube un modal (bottom sheet) con el formulario del apartado.

3
Llenar datos

Nombre de la clienta · Teléfono · Anticipo (el saldo que paga hoy) · Días de vigencia (default 30) · Notas. El monto "Resta por pagar" se calcula en tiempo real.

4
Confirmar → folio generado

El apartado se guarda con folio APT-XXXXXX. El carrito se limpia y el badge de la pestaña Apartados se actualiza.

Lista de apartados activos

La pestaña muestra tarjetas de todos los apartados activos (incluye vencidos). Cada tarjeta muestra nombre, folio, SKUs de las prendas, Total / Anticipo / Resta y fecha de vencimiento. El badge rojo de la pestaña indica cuántos apartados activos hay.

Detalle y gestión

Al tocar una tarjeta se abre el detalle completo. Desde ahí:

  • ✓ Marcar completado — la clienta regresó, pagó el saldo y se llevó las prendas. (Primero procesar el pago de saldo como venta normal en el POS.)
  • ✕ Cancelar — la clienta desistió o venció el plazo.

App vs Sistema web

Misma base de datos, diferente interfaz

💻 Sistema web — pos.php

  • Layout de sidebar + dos columnas
  • Acceso desde cualquier navegador
  • Todos los módulos del sistema
  • Apartados en botón del topbar → modal
  • Cámara web del navegador para QR
  • Dashboard, reportes y configuración

📱 App Android — pose.php

  • Layout mobile-first con pestañas
  • Solo en dispositivo Android con el APK
  • Solo POS y Apartados
  • Apartados en pestaña dedicada
  • Escáner nativo del teléfono (mejor)
  • Optimizada para uso en tienda
✅ Ambas interfaces comparten la misma base de datos y la misma API. Una venta o apartado registrado desde la app aparece al instante en el sistema web, y viceversa.
